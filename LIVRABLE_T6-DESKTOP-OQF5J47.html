0.1 Introduction

La r√©gression logistique est la m√©thode d‚Äôapprentissage la plus populaire apr√®s la r√©gression lin√©aire.

  • En r√©gression lin√©aire la variable d√©pendante (Y) doit √™tre quantitative.

  • En r√©gression logistique la variable r√©ponse (Y) ne prend que deux valeurs. Elle est dans ce cas cod√©e sous forme d‚Äôune variable binaire(Y=0 ou 1).

Exemples

  • Un client r√©pond favorablement √† une offre promotionnelle (oui ou non).

  • Un client d√©cide de renouveler son contrat de t√©l√©phone cellulaire.

  • Une entreprise fait faillite.

  • Un client fait d√©faut de paiement du cr√©dit hypoth√©caire.

  • Un employ√© quitte la compagnie.

  • Un internaute ach√®te un produit en ligne‚Ä¶

0.2 Objectif

On dispose d‚Äôune variable binaire Y (prend 1 en cas d‚Äôoccurrence de l‚Äô√©v√®nement et 0 si non occurrence) et d‚Äôun ensemble de p pr√©dicteurs X (qualitatifs ou quantitatifs).

Les buts de la r√©gression logistique sont essentiellement les m√™mes que la r√©gression lin√©aire:

  • Comprendre comment les variables ind√©pendantes influencent la variable d√©pendante Y: mod√®le explicatif du comportement de Y.

  • Pr√©voir des valeurs de Y futures √† partir des variables ind√©pendantes: mod√®le pr√©dictif.

0.3 Pourquoi la r√©gression lin√©aire ne marche plus?

Le mod√®le de r√©gression lin√©aire a la forme suivante:

\[ E[Y] = \beta_0 + \beta_1 X_1 + \beta_2 X_2 + \dots + \beta_p X_p \quad \text{o√π } E[Y] \text{ repr√©sente l‚Äôesp√©rance (ou la valeur attendue).} \]

Quand Y est binaire, nous pouvons montrer que l‚Äôesp√©rance de la variable binaire est en fait la probabilit√© que Y=1.

Le mod√®le de r√©gression lin√©aire utilis√© avec une variable binaire devient donc :

\[ P(Y = 1) = \cancel{\beta_0 + \beta_1 X_1 + \beta_2 X_2 + \dots + \beta_p X_p} \]

Probl√®me : le c√¥t√© gauche (P[Y=1]) de ce mod√®le prend des valeurs entre 0 et 1 tandis que le c√¥t√© droit peut prendre n‚Äôimporte quelle valeur entre (‚àí‚àû ùëíùë° + ‚àû). Ce mod√®le est donc inad√©quat et pourrait mener √† des pr√©dictions bizarres. Par exemple, le mod√®le pourrait dire que la probabilit√© que le client quitte est -0.3 ! Or une probabilit√© est toujours positive!

0.4 La solution au probl√®me

Solution: Pour √©viter que les r√©sultats ne fassent pas du sens, on va transformer le c√¥t√© gauche du mod√®le afin que les deux c√¥t√©s puissent prendre les m√™mes valeurs

  • Cet objectif est atteint si on utilise le logit(p): \[ \log \left( \frac{P(Y=1)}{P(Y=0)} \right) = \beta_0 + \beta_1 X_1 + \beta_2 X_2 + \dots + \beta_p X_p \]

  • Ce mod√®le est √©quivalent √†

    \[ \frac{P(Y=1)}{P(Y=0)} = e^{\beta_0 + \beta_1 X_1 + \beta_2 X_2 + \dots + \beta_p X_p} \quad \text{o√π } \frac{P(Y=1)}{P(Y=0)} \text{ est un odds (une cote).} \]

  • La r√©gression logistique est donc bas√©e sur ce mod√®le au lieu du mod√®le de r√©gression lin√©aire.

  • Un odds ou une cote de 4 veut dire que P[ Y=1] est 4 fois plus grande que P[ Y=0] (donc que P[Y=1]=0.8).

  • Cote= 2

    \[ \frac{P(Y=1)}{P(Y=0)} = 2 \implies P(Y=1) = 2 \times P(Y=0) = 2 \times (1 - P(Y=1)) \]

    \[ \Rightarrow P(Y=1) = 0.667 \]

0.5 Complexit√© du mod√®le

Le mod√®le de r√©gression logistique est plus complexe que le mod√®le de r√©gression lin√©aire pour les raisons suivantes:

  • L‚Äôalgorithme pour estimer les param√®tres est plus complexe. Cela peut amener des probl√®mes de convergence ou des messages d‚Äôerreur.

  • Les coefficients sont plus difficiles √† interpr√©ter.

  • La pr√©vision est d√©licate (on doit d√©terminer un point de coupure).

    On abordera tous ces points ult√©rieurement.

0.9 M√©thode d‚Äôestimation des param√®tres

  • Rappel mod√®le: \[ \frac{P(Y=1)}{P(Y=0)} = e^{\beta_0 + \beta_1 X_1 + \beta_2 X_2 + \dots + \beta_p X_p} \]

  • Mod√®le th√©orique (population): \[ \log \left( \frac{P}{1 - P} \right) = \beta_0 + \beta_1 X_1 + \beta_2 X_2 + \dots + \beta_p X_p \]

  • Mod√®le ajust√© (√©chantillon): \[ \log \left( \frac{P}{1 - P} \right) = b_0 + b_1 X_1 + b_2 X_2 + \dots + b_p X_p \]

  • Pour des valeurs de X‚ÇÅ, X‚ÇÇ, ‚Ä¶ X‚Çö donn√©es, la probabilit√© estim√©e de l‚Äôoccurrence de l‚Äô√©v√©nement :

    \[ \hat{p}(Y) = \frac{1}{1 + e^{-(b_0 + b_1 X_1 + b_2 X_2 + \dots + b_p X_p)}} \]

0.9.1 Le choix du point de coupure

  • Le mod√®le ajust√© permet ainsi d‚Äôobtenir des pr√©visions de Y en sp√©cifiant un point de coupure (seuil): \[ \hat{p} = \frac{1}{1 + e^{-(b_0 + b_1 X_1 + b_2 X_2 + \dots + b_p X_p)}} \]

  • Par d√©faut, R choisit un seuil de 0.5.

  • Ce point de coupure peut √™tre chang√© si on n‚Äôaccorde pas le m√™me poids au fait de faussement classer un 0 ou de mal classer un 1

    \[ \text{si } \hat{p} < 0.5 \text{ alors il est plus probable que } \hat{Y} = 0 \]

    \[ \text{si } \hat{p} > 0.5 \text{ alors il est plus probable que } \hat{Y} = 1 \]

  • On utilise r√®gle g√©n√©rale la diff√©rence entre (-2LL) de deux mod√®les pour statuer sur la significativit√© de l‚Äôam√©lioration marginale apport√©e par l‚Äôajout de variables (suppl√©mentaires). C‚Äôest en quelque sorte √©quivalent √† un test sur un sous ensemble de param√®tres.
